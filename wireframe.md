## 2. 모바일 vs 데스크톱 주요 차이점

### 모바일 (320px ~ 768px)
- **헤더**: 로고 좌측, CTA 3개 아이콘만 (텍스트 축약)
- **히어로**: 1컬럼, 큰 헤드라인, CTA 버튼 하나
- **상황 선택기**: 2x2 그리드
- **통원 장점**: 1컬럼 세로 스택
- **프로세스**: 세로 스크롤 타임라인
- **프로그램**: 1컬럼 카드 리스트
- **시설/팀**: 1컬럼 이미지 스택
- **FAQ**: 아코디언 전체 너비
- **하단 고정바**: 항상 보임 (핵심 전환 경로)

### 데스크톱 (1024px+)
- **헤더**: 로고 좌측, CTA 3개 텍스트+아이콘 버튼 (우측 배치)
- **히어로**: 좌측 텍스트 + 우측 이미지, 2컬럼 레이아웃
- **상황 선택기**: 1x4 가로 나열
- **통원 장점**: 3컬럼 그리드
- **프로세스**: 가로 스텝바
- **프로그램**: 2x2 그리드
- **시설/팀**: 가로 갤러리 (3-4장)
- **FAQ**: 2컬럼 (질문 목록 좌측, 답변 우측 - 선택사항)
- **하단 고정바**: 상단 헤더가 충분하므로 스크롤 시에만 표시

## 3. CTA 배치 전략

### 상단 고정 헤더 (모든 화면)
```
┌────────────────────────────────────────┐
│ [로고]        [📞전화][💬카톡][📝상담] │
└────────────────────────────────────────┘
```
- **전화**: `tel:` 링크, 클릭 시 즉시 전화 연결
- **카톡**: 카카오톡 상담 URL 또는 오픈채팅
- **상담신청**: 페이지 하단 폼으로 smooth scroll

### 하단 고정바 (모바일 항상 / 데스크톱 스크롤 시)
```
┌────────────────────────────────────────┐
│  [📞전화하기] [💬카톡상담] [📝상담신청] │
└────────────────────────────────────────┘
```
- 모바일: 화면 하단에 항상 고정 (z-index 높음)
- 데스크톱: 스크롤 다운 시 fade-in, 상단 도달 시 fade-out

### 인라인 CTA (섹션별)
- **히어로**: 큰 primary CTA "상담신청하기"
- **상황 선택기**: 각 버튼 클릭 → 해당 프로그램 페이지로 이동
- **프로그램 카드**: "자세히 보기" 링크
- **FAQ 하단**: "더 궁금하신가요? 전화상담 신청하기"

## 4. 주요 인터랙션

### 스크롤 기반
- 섹션별 fade-in 애니메이션 (IntersectionObserver)
- 프로세스 스텝 순차 활성화

### 클릭/터치
- 상황 선택기 버튼: 호버 시 확대, 클릭 시 페이지 전환
- FAQ 아코디언: 클릭 시 expand/collapse
- 프로그램 카드: 호버 시 shadow 증가, 클릭 시 상세 페이지

### 폼 인터랙션
- 상담신청 폼: 실시간 validation
- 전송 성공 시 thank you 메시지 + 자동 전화 안내

## 5. 성능 최적화 고려사항

- 이미지: WebP 포맷, lazy loading
- CSS: Tailwind CDN (프로덕션에서는 빌드 권장)
- JS: 바닐라 JS, 최소한의 라이브러리
- 폰트: 시스템 폰트 우선 (Pretendard CDN 백업)
